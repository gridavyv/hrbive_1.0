MVP:

Базовая гипотеза:
Менеджеры готовы пользоваться сервисом видеовизиток + LLM‑дайджестов, 
потому что это экономит им время и даёт более качественный шорт‑лист, 
и готовы записывать видео-интро и платить за сервис.


Нож для нарезаная фичей
Если какая‑то фича не отвечает на вопрос:
«Повысит ли это вероятность доказать, что сервис нужен?»
она после эксперимента.


Менеджер:
- Согласие на обработку перс данных
- Авторизация через HH OAuth.
- Выбор 1 вакансии из списка.
- Запись видео‑интро о вакансии. !!! MUST !!!
- Получение дайджестов кандидатов (списком по команде админа).
- Возможность по каждому кандидату нажать: «Звать на интервью» / «Не звать» (результат уходит админу).

Соискатель:
- Поясняющий текст
- Согласие на обработку ПД.
- Просмотр видео‑интро менеджера.
- Запись видео‑визитки.

Функционал бота на бэкграунда:
- Скачиваем описание вакансии (автоматом)
- Формируем критерии поиска с помощью ИИ (админ команда)
- Вытаскиваем неразобранные отклики (админ команда)
- Скачиваем резюме + Высылаем ссылку на бот соискателю (админ команда)
- Анализируем резюме (админ команда)
- Отбираем подходящие резюме с видео (админ команда)
- Высылаем новые подходящие резюме с видео менеджеру (админ команда)

Логирование действий:
- менеджер
    - согласие на обработку перс данных
    - HH Auth
    - выбрал вакансию
    - записал видео-интро
    - пригласил / не пригласил кандидата
- кандидат
    - получил приглашение в бот
    - зашел в бот
    - посмотрел видео-интро
    - записал видео-визитку

Сбор статистики (админ команда)
- менеджеры (ID, имя, вакансия, видео-интро)
- вакансия (всего соискателей, подходящих, у скольки есть видео)
- соискатели (ID, вакансия, видео-визитка)
